# Changelog

## v0.3 — 2026-04-27（晚）

### 新增 `/腳本初稿` 命令

把 brief / 大綱 / 訪談 / reference / 客戶會議結論轉成**可拍的逐場腳本**，包括：

- 已 align 方向（三句話濃縮）
- 標題 + 一句話命題 + 觀眾記得的三句話
- 結構表（每場時間 / 功能 / 主訊息）
- **逐場腳本**（畫面：景別/鏡位/運鏡/具體動作；聲音：環境音/配樂節奏/旁白/字幕；過場設計）
- 旁白稿（含表情標註，可直接給配音）
- Shot List（拍攝隊用的 30 行明細表）
- 拍攝注意事項（10 點現場提醒）
- 後期提示（剪接節奏 / 調色 / 配樂 / 音效 / 字幕）

**測試案例**：鉅陞建設第三方驗屋服務片 90 秒 v1 腳本（基於三版提案 A+B 混搭、4/21 會議結論、名威訪談），已輸出至 `02_第三方驗屋服務片/02_企劃與腳本/`。

## v0.2 — 2026-04-27

### `/拆案` 命令升級

第一次實戰測試（鉅陞建設 2026-04-17 會議逐字稿，1090 行）後的反饋升級：

**新增**
- ✨ **輸入類型辨識**：Email brief / 會議逐字稿 / 對話節錄 / 提案文件，每種有專屬處理規則
- ✨ **多案分析**：客戶提到多支影片時自動切換結構（總綱 + 各 sub-project 簡化拆解 + 整合紅旗）
- ✨ **`--context` 歷史脈絡參數**：讓導演把無法從文字看出來的歷史關係（如「之前合作破裂」）納入分析
- ✨ **報價估算（第 7 段）**：工時、報價區間、急件加成、特殊條款建議

**改進**
- 逐字稿類專屬紅旗檢查項（決策者沒到場、抽象詞氾濫、未處理歷史衝突等）
- 風格規則更明確（拒絕模糊、紅旗 emoji 標註）

**測試案例**
- 鉅陞 2026-04-17 會議逐字稿：兩支影片（第三方驗屋 90 秒急件 + 業績大集合 3-5 分鐘 9 月才能拍），自動切換多案結構，識別出窗口 vs 老闆代理動態，發現「之前博社案被接手」的歷史衝突需追問。
